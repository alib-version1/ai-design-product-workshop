# Prototyping for Non-Developers

You do not need to be a developer to build a prototype today. Here is everything you need to know.

## HTML is just text

It is text that browsers know how to display as a web page. You are not writing software. You are directing an AI tool to write it for you.

## To create a page

Make a new file in VS Code, name it `something.html`, and paste in the code from Copilot.

## To see it

Save the file, then double-click it in your file explorer. It opens in your browser.

## To change it

Edit the file in VS Code (or ask Copilot to change it), save, then refresh your browser.

## If it breaks

It is just text. You can undo (Ctrl+Z or Cmd+Z), or delete the file and start again. You cannot damage anything.

## Links between pages

Links between pages are just like links on a website — clicking "Continue" takes you to the next file. If a button says "Continue" and you want it to go to `question.html`, tell Copilot and it will add the link.

## Your job

Think of yourself as the director. Copilot is the camera operator. You say what each page should show — the heading, the content, the buttons, the flow. It writes the code. You check the result in the browser and direct changes.

Your expertise in user needs, content, and service design is what makes the prototype good. The code is just the medium.
