# Prototyping for Non-Developers

You do not need to be a developer to build a prototype today. Here is everything you need to know.

## Open your AI chat panel

**If you are using AWS Kiro:** Click the chat icon in the sidebar. Kiro has two modes — **Vibe** and **Spec**. Use vibe mode today. It generates code directly from your description. Spec mode creates planning documents first, which slows everything down. Check the mode selector in the chat panel before you start.

**If you are using GitHub Copilot in VS Code:** Press **Ctrl+Shift+I** (Windows/Linux) or **Cmd+Shift+I** (Mac) to open the Copilot Chat panel.

## HTML is just text

It is text that browsers know how to display as a web page. You are not writing software. You are directing an AI tool to write it for you.

## To create a page

Make a new file in your IDE (Kiro or VS Code), name it `something.html`, and accept or paste in the code from the chat.

## To see it

Save the file, then double-click it in your file explorer. It opens in your browser.

## To change it

Edit the file in your IDE, or ask your AI tool to change it in the chat, save, then refresh your browser.

## If it breaks

It is just text. You can undo (Ctrl+Z or Cmd+Z), or delete the file and start again. You cannot damage anything.

## Links between pages

Links between pages are just like links on a website — clicking "Continue" takes you to the next file. If a button says "Continue" and you want it to go to `question.html`, tell your AI tool in the chat and it will add the link.

## Your job

Think of yourself as the director. Your AI tool is the camera operator. You say what each page should show — the heading, the content, the buttons, the flow. It writes the code. You check the result in the browser and direct changes.

Your expertise in user needs, content, and service design is what makes the prototype good. The code is just the medium.
